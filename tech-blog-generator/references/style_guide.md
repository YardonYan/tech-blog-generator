# Style Guide - Technical Writing Anti-Patterns

## Forbidden Phrases (DO NOT USE)

### Pedagogical Tone
- "Let's learn..."
- "I hope this helps"
- "As you know..."
- "Beginners often..."
- "Teacher" / "Student" / "Tutor"
- "Follow along"
- "Step-by-step"

### AI Fluff
- "In today's digital landscape..."
- "Unlock the power of..."
- "The future is bright..."
- "Welcome to..."
- "Exciting journey..."
- "Keep coding!"
- "Happy learning!"
- "Stay tuned!"

### Vague Descriptions
- "efficient" (without quantification)
- "robust" (without explanation)
- "optimized" (without metrics)
- "powerful" (without specifics)

## Required Patterns

### Code Block -> Explanation
Every code block MUST be followed by:
1. What it does (one sentence)
2. Why it's designed that way (one sentence)
3. Input/Output specifics

### Comparisons
When comparing A vs B, always include:
| Aspect | A | B |
|--------|---|---|
| Definition | | |
| Mechanism | | |
| Pros | | |
| Cons | | |
| Use Case | | |

## Voice Guidelines

- Use imperative mood: "Use X instead of Y"
- Use passive sparingly: "The function is called" not "We call the function"
- Address as peer: "The reader should..." not "You should..."
